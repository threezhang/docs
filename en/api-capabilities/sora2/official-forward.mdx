## Introduction

**Official Forward** refers to the solution that directly calls OpenAI's official Sora API and transparently forwards it to users. Unlike the current default "reverse-engineered" solution, the Official Forward plan offers higher stability and more precise instruction following.

<Info>
**What is Official Forward?**

Official Forward is a transparent proxy forwarding service for OpenAI's official API. Your requests are directly forwarded to OpenAI's official servers, enjoying the same service quality and stability as OpenAI official.
</Info>

## Plan Comparison

|--------|------------------|--------------------------------------|

| **Image Quality** | Stable, no blur | Occasional blur |

**How to choose?**

</Tip>

## API Reference

### Basic Information

| Configuration | Value |
|---------------|-------|
| **Base URL** | `https://api.laozhang.ai` |
| **Authentication** | `Authorization: Bearer YOUR_API_KEY` |

### API Endpoints

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/v1/videos` | POST | Create video generation task |
| `/v1/videos/{id}` | GET | Query task status |
| `/v1/videos/{id}/content` | GET | Download generated video |

### Request Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `model` | string | Yes | Model name: `sora-2` or `sora-2-pro` |
| `prompt` | string | Yes | Video description text (max 500 characters) |
| `n_seconds` | integer | No | Video duration: 4, 8, or 12 seconds (default 8) |
| `width` | integer | No | Video width (pixels) |
| `height` | integer | No | Video height (pixels) |
| `n_variants` | integer | No | Number of variants to generate (default 1) |

### Supported Resolutions

**sora-2:**
- 720×1280 (Portrait)
- 1280×720 (Landscape)

**sora-2-pro:**
- 720×1280 (Portrait)
- 1280×720 (Landscape)
- 1024×1792 (Portrait HD)
- 1792×1024 (Landscape HD)

## Complete Examples

### cURL Examples

**Step 1: Create Video Generation Task**

```bash
curl -X POST https://api.laozhang.ai/v1/videos \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "sora-2",
    "prompt": "A golden retriever playing fetch on a sunny beach, waves gently rolling in the background, cinematic lighting",
    "n_seconds": 8,
    "width": 1280,
    "height": 720
  }'
```

**Response Example:**

```json
{
  "id": "video_abc123xyz",
  "object": "video.generation",
  "status": "queued",
  "created_at": 1706745600,
  "model": "sora-2",
  "prompt": "A golden retriever playing fetch on a sunny beach...",
  "n_seconds": 8,
  "width": 1280,
  "height": 720
}
```

**Step 2: Poll Task Status**

```bash
curl https://api.laozhang.ai/v1/videos/video_abc123xyz \
  -H "Authorization: Bearer YOUR_API_KEY"
```

**Response Example (In Progress):**

```json
{
  "id": "video_abc123xyz",
  "object": "video.generation",
  "status": "in_progress",
  "created_at": 1706745600,
  "progress": 45
}
```

**Response Example (Completed):**

```json
{
  "id": "video_abc123xyz",
  "object": "video.generation",
  "status": "completed",
  "created_at": 1706745600,
  "finished_at": 1706745900,
  "generations": [
    {
      "id": "gen_xyz789"
    }
  ]
}
```

**Step 3: Download Video**

```bash
curl https://api.laozhang.ai/v1/videos/video_abc123xyz/content \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -o output.mp4
```

### Python Example

```python
import requests
import time

API_KEY = "YOUR_API_KEY"
BASE_URL = "https://api.laozhang.ai"

headers = {
    "Authorization": f"Bearer {API_KEY}",
    "Content-Type": "application/json"
}

# Step 1: Create video generation task
def create_video(prompt, model="sora-2", n_seconds=8, width=1280, height=720):
    response = requests.post(
        f"{BASE_URL}/v1/videos",
        headers=headers,
        json={
            "model": model,
            "prompt": prompt,
            "n_seconds": n_seconds,
            "width": width,
            "height": height
        }
    )
    response.raise_for_status()
    return response.json()

# Step 2: Poll task status
def poll_status(video_id, timeout=600, interval=10):
    start_time = time.time()
    while time.time() - start_time < timeout:
        response = requests.get(
            f"{BASE_URL}/v1/videos/{video_id}",
            headers=headers
        )
        response.raise_for_status()
        data = response.json()

        status = data.get("status")
        print(f"Status: {status}, Progress: {data.get('progress', 'N/A')}%")

        if status == "completed":
            return data
        elif status == "failed":
            raise Exception(f"Video generation failed: {data.get('error')}")

        time.sleep(interval)

    raise TimeoutError("Video generation timeout")

# Step 3: Download video
def download_video(video_id, output_path="output.mp4"):
    response = requests.get(
        f"{BASE_URL}/v1/videos/{video_id}/content",
        headers=headers
    )
    response.raise_for_status()

    with open(output_path, "wb") as f:
        f.write(response.content)
    print(f"Video saved to: {output_path}")

# Complete workflow
if __name__ == "__main__":
    # Create task
    job = create_video(
        prompt="A golden retriever playing fetch on a sunny beach, waves gently rolling in the background",
        model="sora-2",
        n_seconds=8,
        width=1280,
        height=720
    )
    print(f"Task created: {job['id']}")

    # Wait for completion
    result = poll_status(job["id"])
    print("Video generation completed!")

    # Download video
    download_video(job["id"], "my_video.mp4")
```

### Task Status Reference

| Status | Description |
|--------|-------------|
| `queued` | Task queued, waiting to be processed |
| `in_progress` | Video is being generated |
| `completed` | Generation complete, ready for download |
| `failed` | Generation failed |

<Tip>
**Polling Recommendation**

Video generation typically takes 2-5 minutes. We recommend polling status every 10-20 seconds to avoid excessive requests.
</Tip>

## Important Notes

<Warning>
**Key Limitations**

1. **Async only**: Official Forward does not support sync API, all requests are asynchronous
2. **Pay-as-you-go only**: Must select "Pay-as-you-go" billing mode, "Pay-per-call" is not supported

## FAQ

<AccordionGroup>
  <Accordion title="What's the difference between Official Forward and reverse-engineered?">
    Official Forward directly calls OpenAI's official API with 99.99% stability; reverse-engineered is implemented through reverse engineering the official website, with lower prices but potentially affected stability.
  </Accordion>
  <Accordion title="Why doesn't Official Forward support sync API?">
    OpenAI's official Sora API itself only provides async calling method, so Official Forward also only supports async calls.
  </Accordion>
  <Accordion title="Why are duration options different?">
    Official Forward supports OpenAI's official 4/8/12 second duration options; reverse-engineered supports the official website interface's 10/15 second durations.
  </Accordion>
</AccordionGroup>
