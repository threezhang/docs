## Start With the Token Group

For GPT Image 2, the **token group** decides which route your request uses. The same `gpt-image-2` model name maps to different routes in the default group and the `Sora2Official` group.

<CardGroup cols={2}>
  <Card title="Default Group: Reverse Routes" icon="rotate-cw">
    Per-call billing. `gpt-image-2` is the reverse ChatGPT Web route, `gpt-image-2-vip` is the reverse Codex route, and both are **$0.03/call**.
  </Card>
  <Card title="Sora2Official: Official Transit" icon="shield-check">
    Usage-based billing with the same pricing as the official API. The model name is still `gpt-image-2`, and the request shape matches the official Images API.
  </Card>
</CardGroup>

<Tip>
Check the token group first, then the model name. Default-group `gpt-image-2` and `Sora2Official` `gpt-image-2` share the same model name but have different routing, billing, and parameter support.
</Tip>

<Card title="Online Test" icon="external-link" href="https://yingtu.ai">
  You can test the output first at [yingtu.ai](https://yingtu.ai), then move the prompt into your API integration.
</Card>

## Route Matrix

| Token group | Model name | Route | Billing | `size` | `quality` | Available endpoints |
| --- | --- | --- | --- | --- | --- | --- |
| Default group | `gpt-image-2` | Reverse ChatGPT Web | **$0.03/call** | Not supported | Not supported | Chat Completions, Images generation, Images edits |
| Default group | `gpt-image-2-vip` | Reverse Codex | **$0.03/call** | Common sizes only, no 4K | Not supported | Chat Completions, Images generation, Images edits |
| `Sora2Official` | `gpt-image-2` | Official-transit API | Same as official pricing | Supported | Supported | Images generation, Images edits |

## Which Route Should You Use?

| Need | Use |
| --- | --- |
| Quick access without size control | Default-group `gpt-image-2` |
| Reverse route with a common explicit image size | Default-group `gpt-image-2-vip` |
| Existing official OpenAI GPT Image 2 code with full parameter compatibility | `Sora2Official` `gpt-image-2` |
| `quality` or more official parameters | `Sora2Official` `gpt-image-2` |

## Base Configuration

All routes use the same OpenAI-compatible gateway URL:

```bash
export LAOZHANG_API_KEY="sk-your-token"
export BASE_URL="https://api.laozhang.ai/v1"
```

<Info>
Do not set the base URL to a model-specific path. The actual route is determined by the token group and the `model` field.
</Info>

## Parameter Support

### Default-group gpt-image-2

Default-group `gpt-image-2` is the reverse ChatGPT Web route. Use it when you want the simplest integration and do not need size control.

- `size`: not supported
- `quality`: not supported
- Price: **$0.03/call**

### Default-group gpt-image-2-vip

Default-group `gpt-image-2-vip` is the reverse Codex route. Use it when you need a common explicit output size on a reverse route.

- `size`: common sizes supported
- `quality`: not supported
- Price: **$0.03/call**

Recommended `size` values:

- `1024x1024`
- `1536x1024`
- `1024x1536`
- `2048x2048`
- `2048x1152`
- `auto`

VIP size notes:

- Prefer the common sizes listed above.
- `2048x2048` and `2048x1152` are 2K-level sizes. Do not apply official 4K size parameters directly to the VIP reverse route.
- `3840x2160` and `2160x3840` currently trigger upstream `invalid_request_error` / `Unknown error` responses on the VIP reverse route, so treat them as unsupported.

<Warning>
`gpt-image-2-vip` is a reverse route, not the official GPT Image 2 API. Use `gpt-image-2` under the `Sora2Official` group when you need `3840x2160`, `2160x3840`, or full official size support.
</Warning>

### Sora2Official-group gpt-image-2

`Sora2Official` `gpt-image-2` is the official-transit API route. Billing is usage-based with the same pricing as the official OpenAI GPT Image 2 API.

Setup:

1. Create a new token in the console.
2. Select the `Sora2Official` group for that token.
3. Keep `model="gpt-image-2"` in the request body.
4. Keep the official request body and replace only the base URL and API key.

This route supports official parameters, including `size`, `quality`, and other parameters supported by the official API.

Common `size` values:

- `1024x1024`
- `1536x1024`
- `1024x1536`
- `2048x2048`
- `2048x1152`
- `3840x2160`
- `2160x3840`
- `auto`

Available `quality` values:

- `low`
- `medium`
- `high`
- `auto`

## Endpoint Support

Default-group reverse routes support both Chat Completions and Images endpoints:

| Need | Endpoint | Notes |
| --- | --- | --- |
| Text-to-image or image-to-image through chat | `/v1/chat/completions` | The image is usually in a Markdown link at `choices[0].message.content` |
| Images API text-to-image | `/v1/images/generations` | Returns `data[0].b64_json` by default, or `data[0].url` |
| Images API image-to-image | `/v1/images/edits` | Multipart local image upload |

The `Sora2Official` official-transit route must use official-style Images API endpoints:

| Need | Endpoint | Notes |
| --- | --- | --- |
| Text-to-image | `/v1/images/generations` | Matches the official Images generation endpoint |
| Image-to-image | `/v1/images/edits` | Matches the official Images edit endpoint |

<Warning>
This page does not document `Sora2Official` `gpt-image-2` as a `/v1/chat/completions` route. For official transit, use `/v1/images/generations` and `/v1/images/edits`.
</Warning>

## Text-to-Image Examples

### Default-group gpt-image-2

Do not pass `size` or `quality`.

```bash
curl "$BASE_URL/images/generations" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $LAOZHANG_API_KEY" \
  -d '{
    "model": "gpt-image-2",
    "prompt": "Create a clean product render of a white ceramic mug on a gray desk, soft natural light, minimal background"
  }'
```

### Default-group gpt-image-2-vip

Pass `size` only. Do not pass `quality`.

```bash
curl "$BASE_URL/images/generations" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $LAOZHANG_API_KEY" \
  -d '{
    "model": "gpt-image-2-vip",
    "prompt": "Create a clean product render of a white ceramic mug on a gray desk, soft natural light, minimal background",
    "size": "1536x1024"
  }'
```

### Sora2Official-group gpt-image-2

Use the official-style Images generation endpoint. You can pass `size` and `quality`.

```bash
curl "$BASE_URL/images/generations" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $LAOZHANG_API_KEY" \
  -d '{
    "model": "gpt-image-2",
    "prompt": "Create a clean product render of a white ceramic mug on a gray desk, soft natural light, minimal background",
    "size": "1536x1024",
    "quality": "high"
  }'
```

### Default-group Chat Completions

Default-group reverse routes can also generate images through `/v1/chat/completions`:

```bash
curl "$BASE_URL/chat/completions" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $LAOZHANG_API_KEY" \
  -d '{
    "model": "gpt-image-2",
    "messages": [
      {
        "role": "user",
        "content": "Create a clean product render of a white ceramic mug on a gray desk, soft natural light, minimal background"
      }
    ],
    "stream": false
  }'
```

## Image-to-Image Examples

### Default-group Images Edits

```bash
curl "$BASE_URL/images/edits" \
  -H "Authorization: Bearer $LAOZHANG_API_KEY" \
  -F "model=gpt-image-2" \
  -F "prompt=Use the provided image as the source. Keep the subject, composition, and text. Only change the sticker to red and add a very thin gold rim to the mug." \
  -F "image=@source.png"
```

### Default-group gpt-image-2-vip Images Edits

```bash
curl "$BASE_URL/images/edits" \
  -H "Authorization: Bearer $LAOZHANG_API_KEY" \
  -F "model=gpt-image-2-vip" \
  -F "prompt=Use the provided image as the source. Keep the subject, composition, and text. Only change the sticker to blue." \
  -F "image=@source.png" \
  -F "size=1024x1024"
```

### Sora2Official-group Images Edits

Official-transit image editing must use `/v1/images/edits`, with official-style parameters.

```bash
curl "$BASE_URL/images/edits" \
  -H "Authorization: Bearer $LAOZHANG_API_KEY" \
  -F "model=gpt-image-2" \
  -F "prompt=Use the provided image as the source. Keep the subject, composition, and text. Only change the sticker to green." \
  -F "image=@source.png" \
  -F "size=1024x1024" \
  -F "quality=high"
```

### Default-group Chat Completions Image-to-Image

Default-group reverse routes can also send a CDN image URL or base64 data URL through `/v1/chat/completions`:

```bash
curl "$BASE_URL/chat/completions" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $LAOZHANG_API_KEY" \
  -d '{
    "model": "gpt-image-2",
    "messages": [
      {
        "role": "user",
        "content": [
          {
            "type": "text",
            "text": "Use the provided image as the source. Keep the subject, composition, and text. Only change the sticker to green."
          },
          {
            "type": "image_url",
            "image_url": {
              "url": "https://example.com/source.png"
            }
          }
        ]
      }
    ],
    "stream": false
  }'
```

## Parse Responses

### Extract a Chat Completions Image URL

```python
import re

content = response["choices"][0]["message"]["content"]
match = re.search(r"!\[[^\]]*\]\((https?://[^)\s]+)\)", content)
image_url = match.group(1) if match else None
```

### Save Images API b64_json

```python
import base64

value = response["data"][0]["b64_json"]
if value.startswith("data:"):
    value = value.split(",", 1)[1]

value += "=" * ((4 - len(value) % 4) % 4)

with open("output.png", "wb") as f:
    f.write(base64.b64decode(value))
```

### Read an Images API URL

```python
image_url = response["data"][0]["url"]
```

## FAQ

<AccordionGroup>
  <Accordion title="Why does gpt-image-2 mean different things on this page?">
    The token group decides the route. Default-group `gpt-image-2` is the reverse ChatGPT Web route. `Sora2Official` `gpt-image-2` is the official-transit API route.
  </Accordion>
  <Accordion title="Can default-group gpt-image-2 accept size?">
    No. Use `gpt-image-2-vip` in the default group when you need size control.
  </Accordion>
  <Accordion title="Can gpt-image-2-vip accept quality?">
    No. `gpt-image-2-vip` supports common `size` values only. Use `gpt-image-2` in the `Sora2Official` group when you need `quality`.
  </Accordion>
  <Accordion title="Can gpt-image-2-vip use 4K sizes?">
    Treat them as unsupported for now. `3840x2160` and `2160x3840` are official GPT Image 2 sizes, but they fail on the `gpt-image-2-vip` reverse route. Use `gpt-image-2` under the `Sora2Official` group when you need 4K output.
  </Accordion>
  <Accordion title="Can Sora2Official use chat/completions?">
    This page does not document the official-transit route as `/v1/chat/completions`. Use `/v1/images/generations` for generation and `/v1/images/edits` for edits.
  </Accordion>
  <Accordion title="How do I migrate official OpenAI code?">
    Create a token under the `Sora2Official` group, keep the official request body, replace the base URL with `https://api.laozhang.ai/v1`, and replace the API key with your LaoZhang API token.
  </Accordion>
  <Accordion title="Why does b64_json decoding fail?">
    The value may include a `data:image/png;base64,` prefix, or it may need padding at the end. Remove the prefix first, then pad with `=` before decoding.
  </Accordion>
</AccordionGroup>
