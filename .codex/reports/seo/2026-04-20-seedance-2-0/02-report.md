## 三、SERP 结构判断

Google is treating `seedance 2.0` as an entity + access + commercial topic, not only a launch topic. The head SERP contains ads, official pages, product pages, knowledge panel, videos, Reddit/community answers, PAA, and social/video aggregation. This means an article that opens with "what is Seedance 2.0" will be too slow unless it immediately answers "where can I use it, can I call the API, how much does it cost, and which route should I choose?" Evidence: `SERP-main`, `SERP-api`, `SERP-pricing`.

For API queries, Google rewards route clarity and current availability. SERP results include official ByteDance/BytePlus/Volcengine-style docs, fal, Replicate, OpenRouter, Kie, PiAPI, GitHub wrappers, and Reddit provider discussions. This is exactly the query shape where LaoZhang can win as a dated route matrix plus working docs, but only if old "not released yet" language is corrected. Evidence: `SERP-api`, `Source-official`, `Owned-blog`.

For pricing/free queries, Google is surfacing conflicting route types: consumer subscriptions, API per-second pricing, provider marketing pages, Reddit estimates, and free-credit pages. The winning page should not pretend there is one price. It should split official/cloud route, LaoZhang API route, third-party provider route, consumer app route, and free/trial route. Evidence: `SERP-pricing`, `Suggest-api`, `Trends-browser`.

For tutorials, Google leans video-first, but the gap is developer-grade examples. YouTube can show how to use a UI; LaoZhang can show submit-poll-download, model ID selection, `fast` vs standard, media reference inputs, failure/retry handling, and cost checks. Evidence: `SERP-tutorial`, `Owned-docs`, `Source-official`.

For Chinese queries, intent is practical and trust-sensitive. `怎么用` wants the entry path; `API 中转` wants an available route but is surrounded by lower-trust proxy content. The right content style is "which route is real, what can be called, what risks apply, and how LaoZhang's route differs," not a generic Chinese translation of the English page. Evidence: `SERP-zh-howto`, `SERP-zh-api-proxy`, `Suggest-api`.

## 四、内容缺口与切入机会

The current strongest gap is not "no content exists." The gap is that existing owned content contains dated March/early-April assumptions while the visible SERP has moved. Old claims such as "API not released yet" now collide with official/platform docs and provider pages visible in April. That gap is dangerous for both SEO and GEO because AI answer engines prefer current, source-backed route summaries. Evidence: `Owned-blog`, `Source-official`, `SERP-api`.

The second gap is route separation. Searchers are mixing ByteDance official, Volcengine/BytePlus, fal/Replicate/OpenRouter/Kie/PiAPI, Runway/CapCut/Dreamina/Jimeng, and LaoZhang API. A high-value page should define these as separate routes on the first screen and then explain what each is good for. Evidence: `SERP-api`, `SERP-pricing`, `SERP-zh-howto`.

The third gap is model-selection clarity. Existing docs expose both `doubao-seedance-2-0-fast-260128` and `doubao-seedance-2-0-260128`, but the search market also asks `Seedance 2.0 vs Seedance 2.0 fast`. Add a compact "which model should I call?" section with latency/cost/quality/use-case framing. Evidence: `Owned-docs`, `Suggest-api`.

The fourth gap is failure and billing policy. Reddit/provider discussions are full of reliability and cost concerns. A LaoZhang page can win by explaining async task flow, failed-generation billing, moderation failure handling, retry budgets, and how to test before scaling. Evidence: `SERP-api`, `SERP-pricing`, `Owned-docs`.

Use it only for upstream visual-reference/keyframe workflows if there is a genuine feature match. Evidence: `Source-official`, `Owned-blog-laozhang`.
