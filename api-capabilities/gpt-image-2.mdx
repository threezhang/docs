## 先确认令牌分组

GPT Image 2 的线路由**令牌分组**决定。同一个 `gpt-image-2` 模型名，在默认分组和 `Sora2Official` 分组下不是同一条线路。

<CardGroup cols={2}>
  <Card title="默认分组：逆向线路" icon="rotate-cw">
    按次扣费。`gpt-image-2` 是逆向 ChatGPT 网页版，`gpt-image-2-vip` 是逆向 Codex，价格都是 **$0.03/次**。
  </Card>
  <Card title="Sora2Official：官转 API" icon="shield-check">
    按量扣费，价格与官方一致。模型名仍然是 `gpt-image-2`，调用方式与官方 Images API 一致。
  </Card>
</CardGroup>

<Tip>
先看令牌分组，再看模型名。默认分组的 `gpt-image-2` 和 `Sora2Official` 分组的 `gpt-image-2` 名字相同，但参数、计费和接口要求不同。
</Tip>

<Card title="在线测试" icon="external-link" href="https://yingtu.ai">
  可以先在 [yingtu.ai](https://yingtu.ai) 在线测试效果，再迁移到 API。
</Card>

## 线路对照

| 令牌分组 | 模型名 | 线路 | 计费 | `size` | `quality` | 可用接口 |
| --- | --- | --- | --- | --- | --- | --- |
| 默认分组 | `gpt-image-2` | 逆向 ChatGPT 网页版 | **$0.03/次** | 不支持 | 不支持 | Chat Completions、Images 生成、Images 编辑 |
| 默认分组 | `gpt-image-2-vip` | 逆向 Codex | **$0.03/次** | 支持常用尺寸，不含 4K | 不支持 | Chat Completions、Images 生成、Images 编辑 |
| `Sora2Official` | `gpt-image-2` | 官转 API | 与官方一致 | 支持 | 支持 | Images 生成、Images 编辑 |

## 怎么选

| 你的需求 | 选择 |
| --- | --- |
| 快速接入，不需要控制尺寸 | 默认分组 `gpt-image-2` |
| 需要逆向线路，并且要指定常用图片尺寸 | 默认分组 `gpt-image-2-vip` |
| 已有官方 OpenAI GPT Image 2 代码，希望参数完全一致 | `Sora2Official` 分组 `gpt-image-2` |
| 需要 `quality` 或更多官方参数 | `Sora2Official` 分组 `gpt-image-2` |

## 基础配置

三条线路都使用同一个 OpenAI 兼容网关地址：

```bash
export LAOZHANG_API_KEY="sk-你的令牌"
export BASE_URL="https://api.laozhang.ai/v1"
```

<Info>
不要把 base URL 写成模型路径。实际线路由令牌分组和请求里的 `model` 字段共同决定。
</Info>

## 参数支持

### 默认分组 gpt-image-2

默认分组的 `gpt-image-2` 是逆向 ChatGPT 网页版线路，适合不需要尺寸控制的快速接入。

- 不支持 `size`
- 不支持 `quality`
- 价格：**$0.03/次**

### 默认分组 gpt-image-2-vip

默认分组的 `gpt-image-2-vip` 是逆向 Codex 线路，适合需要指定常用输出尺寸的逆向接入。

- 支持常用 `size`
- 不支持 `quality`
- 价格：**$0.03/次**

建议使用的 `size`：

- `1024x1024`
- `1536x1024`
- `1024x1536`
- `2048x2048`
- `2048x1152`
- `auto`

VIP 尺寸说明：

- 优先使用上面的常用尺寸。
- `2048x2048` 和 `2048x1152` 属于 2K 级别；不要把官方 4K 参数直接套到 VIP 逆向线路。
- 当前 `3840x2160` 和 `2160x3840` 在 VIP 逆向线路上会触发上游 `invalid_request_error` / `Unknown error`，按不支持处理。

<Warning>
`gpt-image-2-vip` 是逆向线路，不等同于官方 GPT Image 2 API。需要 `3840x2160`、`2160x3840` 或完整官方尺寸能力时，请使用 `Sora2Official` 分组的官转 `gpt-image-2`。
</Warning>

### Sora2Official 分组 gpt-image-2

`Sora2Official` 分组的 `gpt-image-2` 是官转 API 线路，按量扣费，价格与官方 OpenAI GPT Image 2 API 一致。

使用方式：

1. 在控制台创建新令牌。
2. 令牌分组选择 `Sora2Official`。
3. 请求体继续使用 `model="gpt-image-2"`。
4. 保留官方请求体，只替换 base URL 和 API Key。

支持官方参数，包括 `size`、`quality` 以及官方 API 支持的其他参数。

常用 `size`：

- `1024x1024`
- `1536x1024`
- `1024x1536`
- `2048x2048`
- `2048x1152`
- `3840x2160`
- `2160x3840`
- `auto`

`quality` 可用值：

- `low`
- `medium`
- `high`
- `auto`

## 接口支持

默认分组逆向方案同时支持对话接口和 Images 接口：

| 需求 | 接口 | 说明 |
| --- | --- | --- |
| 对话方式文生图或图改图 | `/v1/chat/completions` | 图片通常在 `choices[0].message.content` 的 Markdown 链接里 |
| Images API 文生图 | `/v1/images/generations` | 默认返回 `data[0].b64_json`，也可返回 `data[0].url` |
| Images API 图改图 | `/v1/images/edits` | multipart 上传本地图片 |

`Sora2Official` 官转方案必须使用官方一致的 Images API：

| 需求 | 接口 | 说明 |
| --- | --- | --- |
| 文生图 | `/v1/images/generations` | 和官方 Images 生成接口一致 |
| 图改图 | `/v1/images/edits` | 和官方 Images 编辑接口一致 |

<Warning>
`Sora2Official` 官转 `gpt-image-2` 不按 `/v1/chat/completions` 文档化。需要官转时，请使用 `/v1/images/generations` 和 `/v1/images/edits`。
</Warning>

## 文生图示例

### 默认分组 gpt-image-2

不要传 `size` 或 `quality`。

```bash
curl "$BASE_URL/images/generations" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $LAOZHANG_API_KEY" \
  -d '{
    "model": "gpt-image-2",
    "prompt": "生成一张白色陶瓷马克杯放在灰色桌面上的产品图，柔和自然光，简洁背景"
  }'
```

### 默认分组 gpt-image-2-vip

只传 `size`，不要传 `quality`。

```bash
curl "$BASE_URL/images/generations" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $LAOZHANG_API_KEY" \
  -d '{
    "model": "gpt-image-2-vip",
    "prompt": "生成一张白色陶瓷马克杯放在灰色桌面上的产品图，柔和自然光，简洁背景",
    "size": "1536x1024"
  }'
```

### Sora2Official 分组 gpt-image-2

使用官方一致的 Images 生成接口，可以传 `size` 和 `quality`。

```bash
curl "$BASE_URL/images/generations" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $LAOZHANG_API_KEY" \
  -d '{
    "model": "gpt-image-2",
    "prompt": "生成一张白色陶瓷马克杯放在灰色桌面上的产品图，柔和自然光，简洁背景",
    "size": "1536x1024",
    "quality": "high"
  }'
```

### 默认分组 Chat Completions

默认分组逆向方案也可以用 `/v1/chat/completions` 文生图：

```bash
curl "$BASE_URL/chat/completions" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $LAOZHANG_API_KEY" \
  -d '{
    "model": "gpt-image-2",
    "messages": [
      {
        "role": "user",
        "content": "生成一张白色陶瓷马克杯放在灰色桌面上的产品图，柔和自然光，简洁背景"
      }
    ],
    "stream": false
  }'
```

## 图改图示例

### 默认分组 Images Edits

```bash
curl "$BASE_URL/images/edits" \
  -H "Authorization: Bearer $LAOZHANG_API_KEY" \
  -F "model=gpt-image-2" \
  -F "prompt=使用提供的图片作为源图。保留主体、构图和文字，只把杯身贴纸改成红色，并给杯口加一条很细的金色边。" \
  -F "image=@source.png"
```

### 默认分组 gpt-image-2-vip Images Edits

```bash
curl "$BASE_URL/images/edits" \
  -H "Authorization: Bearer $LAOZHANG_API_KEY" \
  -F "model=gpt-image-2-vip" \
  -F "prompt=使用提供的图片作为源图。保留主体、构图和文字，只把贴纸改成蓝色。" \
  -F "image=@source.png" \
  -F "size=1024x1024"
```

### Sora2Official 分组 Images Edits

官转图改图必须使用 `/v1/images/edits`，参数按官方 API 写法。

```bash
curl "$BASE_URL/images/edits" \
  -H "Authorization: Bearer $LAOZHANG_API_KEY" \
  -F "model=gpt-image-2" \
  -F "prompt=使用提供的图片作为源图。保留主体、构图和文字，只把贴纸改成绿色。" \
  -F "image=@source.png" \
  -F "size=1024x1024" \
  -F "quality=high"
```

### 默认分组 Chat Completions 图改图

默认分组逆向方案还支持通过 `/v1/chat/completions` 传 CDN 图片地址或 base64 data URL：

```bash
curl "$BASE_URL/chat/completions" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $LAOZHANG_API_KEY" \
  -d '{
    "model": "gpt-image-2",
    "messages": [
      {
        "role": "user",
        "content": [
          {
            "type": "text",
            "text": "使用提供的图片作为源图。保留主体、构图和文字，只把贴纸改成绿色。"
          },
          {
            "type": "image_url",
            "image_url": {
              "url": "https://example.com/source.png"
            }
          }
        ]
      }
    ],
    "stream": false
  }'
```

## 返回结果解析

### 提取 Chat Completions 图片 URL

```python
import re

content = response["choices"][0]["message"]["content"]
match = re.search(r"!\[[^\]]*\]\((https?://[^)\s]+)\)", content)
image_url = match.group(1) if match else None
```

### 保存 Images API 的 b64_json

```python
import base64

value = response["data"][0]["b64_json"]
if value.startswith("data:"):
    value = value.split(",", 1)[1]

value += "=" * ((4 - len(value) % 4) % 4)

with open("output.png", "wb") as f:
    f.write(base64.b64decode(value))
```

### 读取 Images API 的 URL

```python
image_url = response["data"][0]["url"]
```

## 常见问题

<AccordionGroup>
  <Accordion title="为什么都是 gpt-image-2，但说明不一样？">
    因为线路由令牌分组决定。默认分组的 `gpt-image-2` 是逆向 ChatGPT 网页版；`Sora2Official` 分组的 `gpt-image-2` 是官转 API。
  </Accordion>
  <Accordion title="默认分组的 gpt-image-2 能传 size 吗？">
    不支持。默认分组里需要尺寸参数时，用 `gpt-image-2-vip`。
  </Accordion>
  <Accordion title="gpt-image-2-vip 能传 quality 吗？">
    不支持。`gpt-image-2-vip` 只支持常用 `size`。如果需要 `quality`，请使用 `Sora2Official` 分组的 `gpt-image-2`。
  </Accordion>
  <Accordion title="gpt-image-2-vip 能用 4K 尺寸吗？">
    当前按不支持处理。`3840x2160` 和 `2160x3840` 是官方 GPT Image 2 可用尺寸，但在 `gpt-image-2-vip` 逆向线路上会请求失败。需要 4K 输出时，使用 `Sora2Official` 分组的官转 `gpt-image-2`。
  </Accordion>
  <Accordion title="Sora2Official 官转能用 chat/completions 吗？">
    这页不把官转线路作为 `/v1/chat/completions` 来文档化。官转生图使用 `/v1/images/generations`，官转图改图使用 `/v1/images/edits`。
  </Accordion>
  <Accordion title="官转 API 要怎么迁移官方代码？">
    创建 `Sora2Official` 分组令牌后，保留官方请求体，把 base URL 改为 `https://api.laozhang.ai/v1`，把 API Key 改成 LaoZhang API 令牌。
  </Accordion>
  <Accordion title="为什么 b64_json 解码失败？">
    常见原因是返回值带了 `data:image/png;base64,` 前缀，或者末尾缺少 padding。先去掉前缀，再补齐 `=` 后再做 base64 解码。
  </Accordion>
</AccordionGroup>
